# 빠른 시작 가이드

## 1단계: 프로젝트 열기

압축 파일을 해제한 후:

```bash
open CodeComplexityAnalyzer.xcodeproj
```

## 2단계: 빌드 설정 확인

Xcode에서:
- 타겟: "CodeComplexityAnalyzer" 선택
- Destination: "My Mac" 선택
- Signing & Capabilities 탭에서 Team 설정 (선택사항)

## 3단계: 실행

- ⌘+R 누르기 또는
- Product → Run 메뉴 선택

## 4단계: 프로젝트 분석

1. 앱 실행 시 자동으로 폴더 선택 창이 나타남
2. 분석할 Xcode 프로젝트 폴더 선택
3. 잠시 기다리면 분석 완료

## 주요 UI 요소

### 상단 요약 바
- 총 파일 수
- 총 라인 수
- 총 함수 수
- 평균 복잡도
- 가장 복잡한 파일

### 검색 및 정렬
- 🔍 검색창: 파일 이름으로 검색
- 정렬 옵션: 복잡도/라인수/함수수/이름

### 파일 목록
- 각 행을 클릭하면 상세 정보 표시
- 색상으로 복잡도 수준 표시:
  - 🟢 녹색: 낮음
  - 🟡 노란색: 보통
  - 🟠 주황색: 높음
  - 🔴 빨간색: 매우 높음

### 상세 정보
파일을 클릭하면:
- 타입 구성 (Class, Struct, Enum, Protocol)
- 코드 구성 (Properties, Functions, 순환 복잡도)
- 복잡도 게이지

## 팁

### 복잡도가 높은 파일 찾기
1. 정렬을 "복잡도"로 설정
2. 상단에 가장 복잡한 파일들이 표시됨
3. 빨간색으로 표시된 파일 우선 리팩토링

### 큰 파일 찾기
1. 정렬을 "라인 수"로 변경
2. 300줄 이상의 파일은 분할 고려

### 함수가 많은 파일 찾기
1. 정렬을 "함수 수"로 변경
2. 20개 이상의 함수가 있는 파일은 책임 분리 고려

### 특정 파일 빠르게 찾기
1. 검색창에 파일 이름 입력
2. 실시간으로 필터링됨

## 문제 해결

### "폴더를 열 수 없습니다" 오류
- macOS 보안 설정에서 앱 권한 확인
- System Settings → Privacy & Security → Files and Folders

### 분석이 너무 오래 걸림
- 대용량 프로젝트(1000+ 파일)는 1-2분 소요될 수 있음
- Pods, DerivedData 폴더는 자동 제외됨

### 일부 파일이 누락됨
- .swift 파일만 분석됨
- Objective-C 파일(.m, .h)은 지원하지 않음

## 분석 결과 활용

### 리팩토링 계획 수립
```
1. 복잡도 300+ 파일 → 즉시 분할
2. 복잡도 150-300 파일 → 다음 스프린트에 리팩토링
3. 복잡도 50-150 파일 → 모니터링
4. 복잡도 50 미만 → 유지
```

### 코드 리뷰 가이드
```
- 복잡도 높음/매우높음: 30분 이상 리뷰
- 복잡도 보통: 15분 리뷰
- 복잡도 낮음: 10분 리뷰
```

### PR 체크리스트
```
[ ] 새 파일의 복잡도가 150 이하인가?
[ ] 수정된 파일의 복잡도가 증가하지 않았는가?
[ ] 함수 개수가 20개를 초과하지 않는가?
```

## 고급 활용

### 정기적인 복잡도 추적
- 매주/매월 프로젝트 분석
- 복잡도 추이 엑셀로 기록
- 목표: 평균 복잡도 100 이하 유지

### 팀 코딩 컨벤션 설정
```swift
// 목표 가이드라인
- 파일당 최대 300줄
- 함수당 최대 30줄
- 파일당 최대 복잡도 150
```
